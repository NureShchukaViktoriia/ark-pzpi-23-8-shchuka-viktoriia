2 ЗАВДАННЯ
     Завданням даної роботи є ознайомлення з основними принципами оформлення та структурування програмного коду мовою програмування Go.
     Для досягнення поставленої мети необхідно виконати такі завдання:
 * розглянути основні правила оформлення функцій і методів у мові програмування Go;
 * проаналізувати вимоги до форматування коду, зокрема відступів, дужок і вирівнювання виразів;
 * ознайомитися з принципами автоматичного форматування коду за допомогою інструмента gofmt;
 * дослідити правила іменування файлів та організації структури проєкту в Go;
 * навести приклади коду до та після застосування правил оформлення;
 * сформулювати основні рекомендації щодо покращення читабельності та підтримуваності Go-коду.


3 ОПИС ВИКОНАНОЇ РОБОТИ
     3.1 Вступ
     Тема моєї доповіді — правила оформлення програмного коду.
Мова програмування Go (Golang).
     Go — це статично типізована, компільована мова програмування, розроблена компанією Google. Вона була створена з акцентом на простоту, надійність, ефективність і легкість сумісності з високонавантаженими розподіленими системами та багатопотоковістю.
     Go — це нова мова програмування. Хоча вона запозичує ідеї з існуючих мов, вона має незвичайні властивості, які відрізняють ефективні програми на Go від програм, написаних на її родичах. При цьому прямий переклад програми на C++ або Java на Go навряд чи дасть задовільний. З іншого боку, розгляд проблеми з точки зору Go може призвести до успішної, але зовсім іншої програми. Іншими словами, щоб добре писати на Go, важливо розуміти її властивості та ідіоми. Також важливо знати встановлені правила програмування на Go, такі як іменування, форматування, побудова програм тощо.
     3.2 Загальні правила оформлення коду
     Загальні правила оформлення коду:
* вибір зрозумілих імен для змінних, функцій, класів;
* коментарі до коду: коли і як їх використовувати;
* структурування коду (використання відступів, порожніх рядків для покращення читабельності).
     У Go використовується незвичайний підхід і дозволяється машині вирішувати більшість проблем форматування. Програма gofmt (також доступна як go fmt, яка працює на рівні пакета, а не на рівні вихідного файлу) зчитує програму Go та видає вихідний код у стандартному стилі відступів та вертикального вирівнювання, зберігаючи та, за необхідності, переформатуючи коментарі. 
     Наприклад, немає потреби витрачати час на вишикування коментарів у полях структури. Gofmt зробить це за вас. 
     Враховуючи оголошення:
1  type T struct { 
2      name string // ім'я об'єкта 
3      value int // його значення 
4  }
gofmt вирівняє стовпці:
1  type T struct {
2      name    string // name of the object
3      value   int    // its value
4  }
   Залишилися деякі деталі форматування:
     Для відступів використовується табуляція і gofmt генерує їх за замовчуванням. Пробіли використовуються лише за необхідності.
     У Go немає обмеження довжини рядка. Якщо рядок здається занадто довгим, перенесіть його та додайте додатковий відступ табуляцією.
     Go потребує менше дужок, ніж C та Java: керуючі структури ( if, for, switch) не мають дужок у своєму синтаксисі. 

     3.3 Оформлення функцій та методів
     У мові Go особлива увага приділяється простоті та читабельності коду. Тому функції та методи мають бути максимально зрозумілими за структурою, назвою та поведінкою. Основні правила оформлення спрямовані на те, щоб код залишався лаконічним, передбачуваним і легко підтримуваним у великих командах.
     
     3.3.1 Чіткість та стислість функцій
     У Go функції повинні виконувати одну чітку дію. Чим менше логіки всередині — тим простіше зрозуміти, протестувати й повторно використати функцію. Розбиття великих функцій на менші не лише покращує структуру, а й запобігає дублюванню коду.
     Приклад (надто велика функція):
1  func ProcessUser(name string, age int) {
2      fmt.Println("Validating...")
3      if age < 18 {
4          fmt.Println("Too young")
5          return
6      }
7      fmt.Println("Saving:", name)
8  }
     Кращий варіант:
1  func validateAge(age int) bool {
2      return age >= 18
3  }
4  
5   func ProcessUser(name string, age int) {
6      if !validateAge(age) {
7          fmt.Println("Too young")
8          return
9      }
10    fmt.Println("Saving:", name)
11 }
     3.3.2 Відповідність імені функції її призначенню
     Назва функції має чітко пояснювати, що саме вона робить. У Go імена пишуться в CamelCase, а якщо функція експортується — починаються з великої літери. Добра назва усуває потребу в додаткових коментарях.
     Погані назви:
1  func DoIt() {}
2  func Handle() {}
     Гарні назви, що відображають суть:
1  func LoadConfig() {}
2  func CalculateDistance(a, b Point) float64 {}
3  func SendEmail(to string, msg string) error {}

     3.3.3 Використання однакових шаблонів для схожих функцій
     Go заохочує використання уніфікованих підходів: якщо функції роблять схожі дії, вони мають бути оформлені за однаковим шаблоном. Це допомагає програмістам швидко зрозуміти код і передбачити поведінку функцій.
     Типові шаблони в Go:
* повернення (value, error);
* використання зрозумілої структури назв NewType(), LoadX(), SaveX();
* однакова послідовність параметрів.
     Приклад уніфікованих функцій:
1  func LoadUser(id int) (User, error) {
2      // ...
3  }
4  
5  func LoadProduct(id int) (Product, error) {
6      // ...
7  }
     Оформлення функцій у Go базується на простоті: короткі функції, чіткі назви та однакові шаблони. Такі правила допомагають створювати легко читаємий код і значно спрощують підтримку програм у майбутньому.
     
     3.4 Форматування коду
     У Go форматування коду є стандартизованим і фактично автоматизованим. Це забезпечує однаковий стиль у всіх проєктах і дозволяє програмістам зосередитися на логіці, а не на стилі. Основним інструментом є команда gofmt, яка автоматично виправляє відступи, розташування дужок та вирівнювання.
     3.4.1 Відступи
     Go використовує табуляцію, а не пробіли. Стандарт “gofmt” автоматично розставляє потрібні відступи, тому розходжень між командами розробників фактично немає. Вкладені блоки коду завжди мають чітку схему відступів.
     Приклад:
1  func Sum(a, b int) int {
2      result := a + b
3      return result
4  }
     Важливо: 1 рівень вкладеності = 1 табуляція; пробіли вручну не застосовуються для відступів.
     3.4.2 Правила використання круглих і фігурних дужок
     У Go дужки мають строге розташування.
Фігурні дужки {}:
* відкриваюча дужка ставиться на тому ж рядку, що й ключове слово (func, if, for, struct);
* закриваюча дужка розміщується окремим рядком під тим самим відступом, що й початок конструкції.
     Приклад:
1  func PrintValues(nums []int) {
2      for _, v := range nums {
3         fmt.Println(v)
4      }
5  }
     Помилковий стиль (неприйнятний у Go):
1  func Test()
2  {
3      println("bad")
4  }
     Круглі дужки () використовуються тільки у випадках, де це потрібно синтаксисом:
* параметри функцій;
* умови в if, for, але зайві дужки не рекомендуються.
     Go уникає зайвої складності, наприклад умова не повинна містити додаткових дужок:
     Добре:
1  if x > 10 {
2  }
     Погано:
1  if (x > 10) {
2  }
     3.4.3 Вирівнювання операторів та виразів
     Мова програмування Go не рекомендує складні схеми вирівнювання вручну. Вирівнювати змінні чи оператори “у стовпчик” не потрібно — інструмент gofmt сам вирішує, де це доречно.
     Проте Go дозволяє вирівнювати елементи у літералах та оголошеннях, але лише так, як це робить gofmt.
     Правильно (автоматичне форматування):
1  var (
2      width  = 10
3      height = 20
4  )
     Неправильно (ручне вирівнювання пробілами):
1  var (
2      width      = 10
3      height     = 20
4      something  = 30
5  )
     У Go розробник не витрачає час на ручне “красиве” форматування — за це відповідає інструмент.
     Форматування коду в Go майже повністю стандартизоване: використовується табуляція, чітке розміщення фігурних дужок та автоматичне вирівнювання виразів. Завдяки gofmt стиль залишається єдиним і не потребує ручного налаштування, що робить Go одним з найпослідовніших мов у плані оформлення коду.
     3.5 Приклад коду до та після застосування правил оформлення
     Приклад коду до форматування (неоформлений фрагмент):
 1  package main
 2  import "fmt"
 3  func main(){
 4  x:=10
 5  if(x>5){
 6  fmt.Println("Greater")
 7  }else{
 8  fmt.Println("Smaller")
 9  }
10  for i:=0;i<3;i++{
11  fmt.Println(i)
12  }}
     Проблеми:
* відсутні відступи;
* дужки розташовані некоректно;
* круглі дужки в умові if зайві;
* погана читабельність;
* оператори записані без пробілів;
     Той самий код після форматування (стандарт Go + gofmt):
 1  package main
 2  
 3  import "fmt"
 4
 5  func main() {
 6      x := 10
 7
 8      if x > 5 {
 9          fmt.Println("Greater")
10      } else {
11          fmt.Println("Smaller")
12      }
13 
14      for i := 0; i < 3; i++ {
15          fmt.Println(i)
16      }
17  }
     Пояснення змін:
а) Відступи:
   * вкладені блоки (if, else, for) мають 1 табуляцію;
   * код став структурований та легко читається.
     б) Дужки:
* фігурна дужка { перенесена на той самий рядок, що й func, if, for — стандарт Go;
* закриваюча дужка стоїть на окремому рядку на рівні з початком блоку;
* круглі дужки в if(x>5) видалено, оскільки в Go це не прийнято.
в) Пробіли навколо операторів:
     Додано пробіли у виразах:
x:=10 ? x := 10
i:=0; i<3; i++ ? i := 0; i < 3; i++
     г) Порожні рядки для логічних блоків:
     Додано пусті рядки між оголошеннями та конструкціями для кращої читабельності.
     д) Вирівнювання коду відповідно до gofmt:
     Весь код приведено до стандартного стилю без довільних пробілів і неправильних переносів.
     Приклад демонструє, що Go має чіткі правила оформлення коду — від розташування дужок до використання відступів. gofmt гарантує однаковий стиль у всіх файлах, тому відформатований код стає значно читабельнішим, структурованішим і професійним.
     3.6 Оформлення файлів та структури проєкту
     У Go велика увага приділяється простоті й передбачуваності структури проєкту. Правильне оформлення файлів і директорій робить код легшим у навігації, спрощує імпорт пакетів і полегшує підтримку. Більшість рекомендацій ґрунтується на усталених практиках спільноти та офіційних рекомендаціях Go.
     3.6.1 Іменування файлів
     Основні правила:
* імена файлів у Go пишуться маленькими літерами, без пробілів;
* використовуються нижні підкреслення лише коли це виправдано (http_server.go, file_utils.go);
* назва файлу повинна відповідати змісту пакета або його логічній частині;
* файли одного пакета повинні мати змістовні та однорідні назви;
* тестові файли завжди закінчуються на _test.go.
     Приклади правильних назв файлів:
1  user.go
2  user_service.go
3  auth_handler.go
4  config.go
5  database.go
6  math_utils.go
     Приклад тестового файлу: user_test.go
     Погані приклади:
1  UserFunctions.go
2  miscStuff.go
3  file1.go
4  new.go
     Погано, бо назви не інформативні або порушують стиль.


     3.6.2 Структура директорій у проєкті
     Go має простий, зрозумілий та передбачуваний підхід до організації файлів. Зазвичай структура виглядає так:
Рисунок 3.1 - Структура директорій
     Пояснення ключових директорій:
* cmd/ — містить виконувані програми проєкту. Кожна піддиректорія = окремий застосунок;
* internal/ — код, який не можна імпортувати ззовні. Логіка програми, сервіси, модулі. Стандартний спосіб приховати реалізацію;
* pkg/ — код, який може бути використаний іншими проєктами.
Сюди кладуть загальні бібліотеки та інструменти;
* api/ — опис REST/HTTP API, контролерів, Swagger, транспортного рівня;
* configs/ — конфігураційні файли (yaml, json, env);
* go.mod і go.sum — опис модулів та залежностей проєкту.
     Оформлення файлів і структура проєкту в Go будуються на принципах простоти, передбачуваності й мінімалізму. Файли мають зрозумілі назви, а каталоги чітко відображають логічні частини системи. Завдяки цьому проєкти на Go легкі в навігації, швидко масштабується та не потребують складних правил — усе працює за загальноприйнятими стандартами Go.


4 ВИСНОВКИ
     Покращення коду в Go ґрунтується на простоті, структурованості та послідовності. Дотримання стандартів оформлення робить програми більш читабельними, надійними та зручними в підтримці. Основні принципи, яких необхідно дотримуватися:
* чіткість і лаконічність: функції та методи мають виконувати одну конкретну дію. Простий і прямолінійний код легше тестувати й розуміти;
* говорючі назви: назви функцій, файлів і пакетів повинні відображати їхнє призначення. Це зменшує потребу в коментарях і прискорює розуміння логіки;
* єдиний стиль оформлення: використання gofmt забезпечує однакові відступи, дужки та вирівнювання у проєкті. Стандартний стиль усуває суперечки та хаос у форматуванні;
* логічна структура проєкту: чітка організація директорій робить код передбачуваним, а модулі — зрозумілими й добре ізольованими;
* повторюваність та узгодженість: схожі функції мають однакові патерни: порядок параметрів, структура повернення значень, формат обробки помилок. Це значно спрощує читання та підтримку;
* мінімізація зайвого коду: у Go не вітається надмірна складність: зайві дужки, оператори, великі функції чи громіздкі конструкції. Менше — краще;
* прозора обробка помилок: стандартний підхід (value, error) забезпечує передбачувану поведінку та контроль над помилками.
     Якісний Go-код — це код, який легко читати, просто підтримувати  безпечно розширювати. Дотримання стилю, чітка структура та послідовність у всіх частинах проєкту створюють професійне середовище, де код працює на розробника, а не навпаки.
